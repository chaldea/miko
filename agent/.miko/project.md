# Project Structure & Hosting

## Anatomy of a generated app

Single-platform (desktop) app:

```
MyApp/
├── App.cs            # MikoAppBuilder configuration (title, size, routes, layout)
├── Program.cs        # Entry point — builds the context and runs the desktop host
├── GlobalStyles.cs   # App-wide StyleSheet, built in C#
├── MainLayout.razor  # Default layout wrapping every routed page
├── _Imports.razor    # Shared @using directives for all .razor files
└── Pages/
    └── Home.razor    # A routed page (@page "/")
```

Multiplatform app (shared UI + one thin host per platform):

```
MyApp/
├── MyApp/           # Shared Razor UI (Pages, MainLayout, GlobalStyles, App.cs)
├── MyApp.Desktop/   # Program.cs → RunDesktop() (refs Miko.Windowing)
├── MyApp.Android/   # MainActivity + manifest (refs Miko.Android)
├── MyApp.iOS/       # AppDelegate + Info.plist (refs Miko.iOS)
└── MyApp.Simulator/ # Device preview host
```

The shared project references only `Miko` (+ component packages like `Miko.Ionic`). Each host owns the window/GL surface/native input and drives the render loop.

## `App.cs` — configure with `MikoAppBuilder`

`MikoAppBuilder.CreateDefault()` registers core services and returns a builder:

```csharp
using Miko.DevTools;
using Miko.Hosting;

namespace MyApp;

public static class App
{
    public static MikoAppContext CreateContext()
    {
        var builder = MikoAppBuilder.CreateDefault();

        builder.UseTitle("MyApp");
        builder.UseSize(1024, 768);

        builder.AddDevTools();
        builder.AddStyleSheet(GlobalStyles.Create());

        builder.UseGeneratedRoutes();        // routes from @page directives
        builder.UseDefaultLayout<MainLayout>();
        builder.EnableHotReload();

        return builder.Build();              // returns MikoAppContext
    }
}
```

### Builder reference

| Method | Purpose |
| --- | --- |
| `CreateDefault()` | Static factory registering default services. |
| `UseTitle(string)` | Window / app title. |
| `UseSize(int w, int h)` | Initial viewport size (px). |
| `UseRootComponent(Func<Element>)` | Explicit root component factory (instead of routes). |
| `UseStyleSheets(List<StyleSheet>)` | Replace the stylesheet list. |
| `AddStyleSheet(StyleSheet)` | Append one stylesheet. |
| `UseGeneratedRoutes()` | Wire up routes discovered from `@page` (generated extension). |
| `UseRouter(params Assembly[])` / `UseRouter(Action<Router>)` | Configure routing manually. |
| `UseDefaultLayout<TLayout>()` / `UseDefaultLayout(Type)` | Default layout for routed pages. |
| `UseFonts(Action<FontBuilder>)` | Register custom fonts. |
| `AddGlobalKeyHandler(Func<MikoKey, bool>)` | App-wide key shortcuts (return `true` to consume). |
| `UseLogging(Action<ILoggingBuilder>)` | Configure logging. |
| `AddDevTools()` | Runtime DOM/layout inspector (needs `Miko.DevTools`). |
| `AddIonic()` | Register the Ionic component library (needs `Miko.Ionic`). |
| `EnableHotReload()` | Turn on `.razor` hot reload. |
| `Build()` | Build the platform-agnostic `MikoAppContext`. |

`builder.Services` is an `IServiceCollection` — register your own services (HTTP clients, view models) and `@inject` them into components.

## `Program.cs` — run the host

```csharp
using Miko.Windowing;

public static class Program
{
    [STAThread]
    public static void Main(string[] args)
    {
        var context = App.CreateContext();
        // Initialize hot reload handler generated by Miko.Razor.Compiler.
        MikoHotReloadHandler.Initialize(context.GetHotReloadService());
        context.RunDesktop();
    }
}
```

Per platform, consume the same `MikoAppContext`:

```csharp
context.RunDesktop();                                                   // Miko.Windowing
SetContentView(MikoAndroidApp.CreateView(this, App.CreateContext));     // Miko.Android
Window.RootViewController = new MikoViewController(App.CreateContext()); // Miko.iOS
context.RunSimulator();                                                  // Miko.Simulator
```

## Routing & layouts

- A `.razor` file with `@page "/route"` is a route. Multiple `@page` directives on one file give multiple routes.
- `UseGeneratedRoutes()` discovers them at build time.
- Navigate programmatically: `@inject Miko.Routing.NavigationManager Navigation;` then `Navigation.NavigateTo("/about")`. Read `Navigation.CurrentPath`.
- A **layout** inherits `Miko.Components.LayoutComponentBase` and renders `@Body` where the page goes. Register with `UseDefaultLayout<MainLayout>()`.

## Direct engine API (no Razor)

Build the `Element` tree by hand and drive `MikoEngine`:

```csharp
using Miko.Core;
using Miko.Core.DomElements;
using Miko.Styling;
using Miko.Styling.Selectors;
using Miko.Common;
using SkiaSharp;

var styleSheet = new StyleSheet();
styleSheet.AddRule(new ClassSelector("container"), new Style
{
    Display = Display.Flex,
    FlexDirection = FlexDirection.Column,
    Padding = new Padding(Length.Px(20)),
    BackgroundColor = new Color(245, 245, 245),
});

var root = new DivElement { Class = "container" };
root.AddChild(new H1Element { TextContent = "Hello Miko" });
root.AddChild(new ParagraphElement { TextContent = "Lightweight rendering" });

using var surface = SKSurface.Create(new SKImageInfo(800, 600));
var engine = new MikoEngine();
engine.Initialize(root, new List<StyleSheet> { styleSheet }, surface.Canvas, 800, 600);
engine.Render(surface.Canvas);
```

Incremental update:

```csharp
element.TextContent = "Updated";
engine.InvalidateElement(element);
engine.Update(canvas);   // repaints only dirty regions
```
